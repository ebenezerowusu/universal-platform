# Release Decision Note Template

## Purpose

This template records the decision for a major release window.

## Template

```text
# Release Decision Note: <Release Name>

## Date

YYYY-MM-DD

## Prepared By

Name or team.

## Release Reference

Branch, tag, commit, or build reference.

## Decision

Choose one:

- proceed
- proceed with limitations
- run another rollout wave
- improve further before release
- pause release

## Reason

Short explanation of the decision.

## Evidence Reviewed

- rollout metrics
- support records
- smoke check results
- backup confidence
- known limitations

## Known Limitations

- limitation 1

## Open Concerns

- concern 1

## Actions Before Release

- action 1

## Actions After Release

- action 1

## Owners

- release owner
- support owner
```

## Final Rule

A release decision should be written down before the release window opens.
