# Pilot Readiness Report Template

## Purpose

This template should be copied when preparing the MVP for a controlled pilot organization.

It records what is ready, what is limited, and what risks remain.

## Template

```text
# Pilot Readiness Report: <Pilot Name>

## Date

YYYY-MM-DD

## Prepared By

Name or team.

## Release Candidate

Branch, tag, commit, or build reference.

## Pilot Organization

Organization name or placeholder.

## Summary

Short summary of pilot readiness.

## MVP Areas Included

- Identity
- Organization
- Permission
- Audit
- Module Registry
- Subscription/Feature foundation
- Religious settings
- Members
- Visitors
- Structure
- Attendance
- Basic reports
- Flutter shell
- Religious Flutter workflows

## Areas Explicitly Excluded

- QR attendance
- GPS attendance
- live SMS
- live payments
- full offline sync
- POS/Commerce
- HR/Finance
- advanced Religious modules

## End-to-End Test Result

Passed / Failed / Blocked / Partial

## UAT Result

Passed / Failed / Blocked / Pending

## Critical Issues

- issue 1

## High Issues

- issue 1

## Accepted Limitations

- limitation 1

## Operational Readiness

- backend health verified: yes/no
- database migrations verified: yes/no
- seed data verified: yes/no
- logs reviewed: yes/no
- support process defined: yes/no

## Security Readiness

- authentication reviewed: yes/no
- organization boundary reviewed: yes/no
- permission behavior reviewed: yes/no
- audit behavior reviewed: yes/no
- sensitive data exposure reviewed: yes/no

## Decision

Choose one:

- ready for pilot testing
- ready with accepted limitations
- not ready

## Next Actions

- action 1
```

## Final Rule

A pilot readiness report should be honest. Do not hide limitations or unresolved risks.
