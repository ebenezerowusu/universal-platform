# Sprint 9 Controlled Pilot Plan

## Purpose

This document defines Sprint 9 for controlled pilot launch readiness.

Sprint 9 should prepare the MVP for real pilot usage with clear support, feedback, and issue handling.

## Sprint Goal

Launch the MVP to a controlled pilot group in a way that is safe, observable, and limited in scope.

## Prerequisites

Before Sprint 9 begins:

- MVP release candidate is prepared
- pilot-readiness report exists
- critical issues are resolved
- accepted limitations are documented
- support owner is identified
- pilot organization is selected

## Work Package 1: Pilot Launch Checklist

Prepare a final checklist for:

- pilot organization setup
- admin/owner user setup
- module activation
- permissions and roles
- seed/demo data review
- support contact setup
- known limitations review

## Work Package 2: Pilot Onboarding

Prepare onboarding direction for pilot users:

- what the MVP includes
- what the MVP excludes
- how to log in
- how to select organization
- how to manage members
- how to manage visitors
- how to use structure and attendance
- how to report issues

## Work Package 3: Pilot Support

Prepare support workflows for:

- login issues
- organization access issues
- permission issues
- member/visitor workflow issues
- attendance issues
- data correction requests
- unexpected errors

## Work Package 4: Feedback Capture

Prepare a structured way to collect:

- bugs
- confusing workflows
- missing MVP expectations
- improvement requests
- future feature requests

## Work Package 5: Issue Triage

Define triage levels:

```text
critical - blocks pilot usage
high - serious workflow issue
medium - workaround exists
low - polish or future improvement
```

## Work Package 6: Post-Pilot Review

Prepare post-pilot review template.

The review should answer:

- what worked
- what failed
- what confused users
- what must be fixed before wider launch
- what should stay deferred

## Out of Scope

Do not add:

- new major features
- advanced Religious modules
- unrelated refactors
- provider integrations unless approved
- non-MVP domains

## Completion Standard

Sprint 9 is complete when the team can start a controlled pilot with clear setup, support, feedback, and post-pilot review processes.

## Final Rule

A pilot is not a full launch. Keep the scope controlled and use the pilot to learn safely.
