# Sprint 11 Limited Rollout Roadmap

## Purpose

This document defines Sprint 11 for limited rollout readiness.

Sprint 11 prepares the platform to support a small group of organizations after the first pilot has been completed and reviewed.

## Sprint Goal

Move from one pilot organization to a small controlled group while keeping setup, support, testing, and review repeatable.

## Prerequisites

Before Sprint 11 begins:

- post-pilot hardening is complete
- wider rollout checklist has a positive decision
- known limitations are documented
- support process is active
- onboarding process is repeatable
- release candidate is stable enough for limited rollout

## Work Package 1: Rollout Scope

Define:

- number of organizations to onboard
- user roles to include first
- MVP features included
- MVP exclusions
- rollout duration
- support owner
- success criteria

## Work Package 2: Organization Onboarding

Prepare a consistent setup process for each organization:

- organization setup
- owner/admin setup
- module activation
- plan/features
- starter access profiles
- starter Religious settings
- sample congregation/service/group where appropriate

## Work Package 3: Smoke Tests

Run smoke tests after each organization setup.

Smoke tests should cover:

- login
- organization selection
- Religious dashboard
- member list and create
- visitor list and create
- attendance session list and create
- manual attendance marking where available

## Work Package 4: Support Readiness

Confirm support can handle more than one organization clearly.

Every support item should include organization name, user role, workflow, and severity.

## Work Package 5: Operational Review

Track:

- organizations onboarded
- active pilot users
- issues by severity
- support response patterns
- workflow completion problems
- accepted limitations

## Out of Scope

Do not add:

- public launch features
- unrelated modules
- large feature expansion
- advanced Religious workflows unless approved as rollout-critical

## Completion Standard

Sprint 11 is complete when several organizations can be onboarded using the same process with clear support and smoke testing.

## Final Rule

Limited rollout should prove repeatability. If every organization requires custom manual work, the rollout process is not ready.
