## Documentation Principle

Documentation is not decoration. It is part of the architecture.

Before implementing any feature, contributors must understand:

- The platform vision.
- The platform constitution.
- The separation between Platform Core, Capability Engines, Domains, and Infrastructure Adapters.
- The rules for multi-tenancy, modularity, provider abstraction, and configuration-driven behavior.

This documentation set is the starting foundation. It will evolve before development begins.
