# Initial Role Templates

## Purpose

This document records starter role templates for Universal Platform.

Role templates help new organizations start quickly. They are not permanent hardcoded access rules.

## Principle

A role template is a setup helper.

The Permission Engine remains the source of authorization decisions.

## Platform Templates

### Platform Administrator

Purpose:

- Manage platform-level records and configuration.

Typical responsibility areas:

- organizations
- modules
- subscription plans
- provider configuration
- platform audit records

### Platform Support

Purpose:

- Help with support and operational review.

Typical responsibility areas:

- organization lookup
- module status review
- support diagnostics

## Organization Templates

### Organization Owner

Purpose:

- Manage the full organization account.

Typical responsibility areas:

- organization profile
- users
- roles
- settings
- modules
- subscription visibility

### Organization Administrator

Purpose:

- Manage daily organization operations.

Typical responsibility areas:

- organization information
- organization users
- settings review
- module catalog review

### Finance User

Purpose:

- Manage financial work where allowed.

Typical responsibility areas:

- giving records
- payment records
- financial reports

### Communication User

Purpose:

- Manage communication work where allowed.

Typical responsibility areas:

- announcements
- messages
- campaigns
- delivery reports

## Religious Templates

### Branch Leader

Purpose:

- Manage assigned branch operations.

Typical responsibility areas:

- members
- visitors
- attendance
- groups
- branch reports

### Group Leader

Purpose:

- Manage assigned group operations.

Typical responsibility areas:

- assigned group
- attendance marking
- follow-up tasks

### Care User

Purpose:

- Support care and follow-up operations.

Typical responsibility areas:

- care records
- prayer records
- follow-up tasks

## Customization

Organizations should be able to create custom roles from templates.

Updating a template should not unexpectedly change already-customized organization roles unless a clear sync process is introduced.

## Final Rule

Role templates help setup. Permissions decide actual access.
