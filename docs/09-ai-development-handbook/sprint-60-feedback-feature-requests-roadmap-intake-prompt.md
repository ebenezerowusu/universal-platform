# Sprint 60 Feedback Feature Requests Roadmap Intake Prompt

## Purpose

This document is a copy-paste-ready prompt for a coding agent to begin Sprint 60.

Sprint 60 prepares the Feedback, Feature Requests, and Product Roadmap Intake foundation after product announcements, knowledge base, support operations, data quality, search, workflow, governance, and notification foundations have introduced structured product feedback needs.

## Expected Output

The coding agent should produce a focused feedback and roadmap intake foundation for feedback items, feature requests, idea categories, duplicate grouping, prioritization placeholders, roadmap candidates, support/help/article links, release/announcement links, status responses, and audit-safe product feedback operations.

## Final Rule

Feedback intake should organize customer and user input without promising delivery. Roadmap candidates are reviewable planning signals, not commitments.
