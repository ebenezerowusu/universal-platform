# Sprint 55 Governance Delegated Authority Prompt

## Purpose

This document is a copy-paste-ready prompt for a coding agent to begin Sprint 55.

Sprint 55 prepares the Governance Policy, Delegated Authority, and Approval Mandates foundation after organization network, workflow, permission, finance, HR, data portability, privacy, provider connections, and support operations foundations have introduced authority-decision needs.

## Expected Output

The coding agent should produce a focused governance authority foundation for policy metadata, authority mandates, delegated approvers, approval thresholds, domain action mappings, temporary delegations, conflict warnings, authority decision logs, and audit-safe delegated authority operations.

## Final Rule

Governance authority must complement Permission Engine and Workflow Engine, not replace them. Permission answers whether an actor may access a capability; governance authority defines whether the actor is an approved decision-maker for a sensitive business action.
