# Sprint 64 Implementation Projects Onboarding Delivery Migration Runbooks Prompt

## Purpose

This document is a copy-paste-ready prompt for a coding agent to begin Sprint 64.

Sprint 64 prepares the Implementation Projects, Onboarding Delivery, and Migration Runbooks foundation after customer success, organization lifecycle, import/export, data portability, data quality, workflow, support operations, knowledge base, and product analytics foundations have introduced structured customer onboarding and go-live delivery needs.

## Expected Output

The coding agent should produce a focused implementation delivery foundation for onboarding projects, milestones, checklists, migration runbooks, migration task links, setup tasks, acceptance checks, go-live readiness, implementation risks/issues, customer success/support/help links, and audit-safe delivery operations.

## Final Rule

Implementation delivery must coordinate onboarding and migration work without replacing project management, bypassing data quality checks, or automatically approving go-live. Go-live readiness is a reviewed state, not a system guarantee.
