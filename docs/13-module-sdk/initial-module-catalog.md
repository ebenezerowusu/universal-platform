# Initial Module Catalog

## Purpose

This document defines the first planned module catalog for Universal Platform.

The module catalog helps guide Module Registry implementation, permissions, navigation, subscriptions, and future packaging.

## Platform/Core Modules

### identity

Purpose:

- User identity and authentication foundation.

### organizations

Purpose:

- Tenant and organization management.

### permissions

Purpose:

- Roles, permissions, and scopes.

### audit

Purpose:

- Sensitive action tracking.

### module_registry

Purpose:

- Module installation and activation.

### subscriptions

Purpose:

- Plan, feature, and limit access.

## Capability Engine Modules

### payments

Purpose:

- Payment transactions and provider abstraction.

### sms

Purpose:

- SMS wallet, packages, messages, and provider abstraction.

### notifications

Purpose:

- In-app and channel-based notification coordination.

### storage

Purpose:

- File metadata, access rules, and provider abstraction.

### reporting

Purpose:

- Report definitions, dashboards, and exports.

### workflow

Purpose:

- Tasks, approvals, and configurable processes.

### import_export

Purpose:

- CSV/Excel import and export workflows.

### localization

Purpose:

- Language, region, timezone, currency, and terminology settings.

## Final Rule

The module catalog should grow intentionally. Do not create hidden modules that bypass Module Registry rules.
